# aws-reStart-journey-

My Daily Routine in the AWS re/Start Program
A big part of the AWS re/Start 12 week program is consistency, hands on learning, and practicing cloud engineering skills every single day.
Here’s how I can describe my day to day work in a way that shows discipline, technical growth, and readiness for real world roles:
________________________________________
 Daily Structure
 
<b> 1- Morning Stand Up & Learning Objectives  </b>

Every morning, I start by reviewing the day’s learning goals:
-	Understanding new AWS concepts
-	Reviewing architecture diagrams
-	Preparing questions for group discussions
This helps me develop planning habits similar to Agile teams.
________________________________________
<b> 2- Instructor Led Cloud Training  </b>

I spend a large portion of the day in guided lessons on:
-	AWS core services (EC2, VPC, S3, IAM, Lambda, RDS, CloudWatch)
-	Linux administration
-	Networking fundamentals
-	Security best practices
These sessions help build a strong theoretical foundation before jumping into labs.
________________________________________
<b> 3- Hands On Labs & Practical Cloud Projects </b>

This is the most important and intensive part of the day.
I apply everything I learn by building and deploying cloud resources in AWS:
- Creating EC2 instances & configuring Linux
-	Designing VPC networks (subnets, route tables, NAT, security groups)
-	Deploying web servers
-	Automating tasks using Python
-	Working with S3 buckets, versioning, and lifecycle policies
-	Creating serverless workflows with Lambda & CloudWatch triggers
Hands on practice shows that I can do, not just talk about cloud topics.
________________________________________
<b> 4- Teamwork, Collaboration & Problem Solving </b>

Every day involves group assignments where we:
-	Troubleshoot failures
-	Work through architecture challenges
-	Review each other’s code or configurations
-	Present solutions
This builds strong communication and teamwork skills needed for DevOps & cloud roles.
________________________________________
<b> 5- GitHub & Documentation Practice  </b>

I document labs, write clean explanations, and push updates to GitHub.
________________________________________
<b> 6- Afternoon Review & Practice Questions  </b>

To reinforce learning, I finish the day with:
-	AWS practice quizzes
-	Reviewing CLI commands
-	Practicing Linux tasks
-	Deep dive reading on services
This helps me build consistency and long term retention.
________________________________________

#  Aurora MySQL Lab —  Summary

This lab walked me through the full workflow of deploying an **Amazon Aurora MySQL cluster**, connecting to it from an **EC2 Command Host**, and performing basic SQL operations.  
It helped me understand how Aurora works inside a VPC, how endpoints behave, and how to interact with the database using the MySQL client.

---

##  **Task 1 — Create an Aurora MySQL Cluster**

I started by provisioning an **Aurora (MySQL-Compatible)** database using the **Standard create** workflow in Amazon RDS.

Key configuration choices:
- **Engine:** Aurora MySQL 8.0 (default version)  
- **Template:** Dev/Test  
- **Cluster identifier:** `aurora`  
- **Master user:** `admin` / `admin123`  
- **Instance class:** `db.t3.medium`  
- **Multi‑AZ:** Disabled (not needed for the lab)  
- **VPC:** `LabVPC`  
- **Subnet group:** `dbsubnetgroup`  
- **Security group:** `DBSecurityGroup`  
- **Initial database:** `world`  
- **Encryption & auto‑upgrade:** Disabled  

Once created, the cluster showed an **Available** status and was ready for connection.

##  **Task 2 — Connect to the EC2 Command Host**

To interact with Aurora, I connected to a pre‑provisioned EC2 instance:

1. Opened **EC2 → Instances**  
2. Selected the instance named **Command Host**  
3. Connected using **Session Manager**  

This opened a browser‑based terminal where I could run commands without needing SSH keys.

---

##  **Task 3 — Configure EC2 to Connect to Aurora**

Inside the EC2 terminal, I installed the MariaDB/MySQL client:

```bash
sudo yum install mariadb -y
```

Then I retrieved the **Writer endpoint** from:

**RDS → aurora → Connectivity & security → Endpoints**

```
aurora.cluster-crcrqygathh1.us-west-2.rds.amazonaws.com
```

Finally, I connected to the Aurora database:

```bash
mysql -u admin --password='admin123' -h aurora.cluster-crcrqygathh1.us-west-2.rds.amazonaws.com
```

This opened the MySQL shell:

```
MySQL [(none)]>
```

---

##  **Task 4 — Create a Table, Insert Data, Query Results**

Inside the MySQL shell, I explored and interacted with the `world` database.

### **1. List databases**
```sql
SHOW DATABASES;
```

### **2. Switch to the `world` database**
```sql
USE world;
```

### **3. Create the `country` table**
```sql
CREATE TABLE country (...);
```

### **4. Insert sample records**
Five countries were inserted using `INSERT` statements.

### **5. Query the data**
```sql
SELECT * FROM country
WHERE GNP > 35000 AND Population > 10000000;
```

**Result:**  
Two matching rows — **Australia** and **Thailand**.
